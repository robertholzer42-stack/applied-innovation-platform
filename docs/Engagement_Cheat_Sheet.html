Coordinated Engagement Cheat Sheet

Applied Innovation Platform - 8-Stage Pipeline
1
Intake
Navigator
Say this
"Navigator, run intake for a new engagement. Client: [name/industry]. Challenge: [what they're trying to solve]. Context: [why now, what's been tried, constraints]."

Navigator will ask discovery questions, frame the focal question, and recommend Quick/Standard/Deep scope with agent priorities. The engagement brief is a LIVING document. The Conductor updates it after each stage when findings change the picture. Downstream agents always read the latest version.

Optional: "Also run the innovation maturity assessment."

Output: Living Engagement Brief + Focal Question + Recommended Agent Lineup
2.0
Research Foundation
Conductor

Conductor builds a shared fact base so all Tier 2 agents work from the same data. Skip for quick scans.

Automatic (Conductor handles this)
The Conductor researches: market size, key players, regulatory context, recent developments (last 12 months), technology state of the art.
Output: research-foundation.md (shared by all downstream agents)
2
Core Analysis (run in parallel on shared data)
Scout + Empathy + Architect
Scout (future thinking)
"Scout, run a [standard/deep] foresight analysis on [focal question]. Time horizon: [X] years. Here's the engagement context: [paste brief]. Use the research foundation."
Empathy (design thinking)
"Empathy, map the human needs and user experience for [focal question]. Who are the key stakeholders? What are their pain points? Here's the context: [paste brief]."
Architect (systems thinking)
"Architect, map the system dependencies and dynamics for [focal question]. What are the feedback loops? Where are the leverage points? Context: [paste brief]."
Output: Three independent analysis briefs (foresight, human, systems)
2.5
Preliminary Scoring
Scorekeeper (first pass)

Scorekeeper produces early DVFA scores based on Tier 2 evidence. Flags dimensions below 2.0 for extra attention.

Automatic (Conductor triggers this)
"Scorekeeper, run preliminary DVFA scoring based on Scout, Empathy, and Architect outputs. Flag any dimension with low confidence and generate evidence requests for downstream agents."
Output: Preliminary DVFA scorecard (marked PRELIMINARY) + evidence request list
3
Intersection Synthesis (receives Stage 2 outputs + preliminary DVFA)
Visionary + Integrator + Sentinel
Visionary (future + design)
"Visionary, here are Scout's forecasts and Empathy's personas. How do these users evolve under these scenarios? What should we design for? [paste key findings]"
Integrator (design + systems)
"Integrator, here are Empathy's journey map and Architect's system analysis. Where does the experience break because of system constraints? What's the path to scale? [paste key findings]"
Sentinel (future + systems)
"Sentinel, here are Scout's scenarios and Architect's system map. Stress-test the strategy. What breaks first? What assumptions are load-bearing? [paste key findings]"
Output: Three intersection briefs (future-design, design-systems, future-systems)
4
Operational Context (run as needed)
Radar + Banker + Scorekeeper + Bridge
Run whichever Navigator recommended. Common prompts:
"Radar, what's the competitive landscape for [domain]?"
"Banker, how does this fit our innovation portfolio?"
"Scorekeeper, score this opportunity using DVFA. Here are the inputs: [summary from each agent]."
"Bridge, assess change readiness for [initiative]. Here's the org context: [details]."

After Radar/Banker/Bridge complete, Scorekeeper runs the final DVFA pass with evidence from all agents. Shows preliminary-to-final delta with explanations.

Output: Competitive intel, portfolio fit, change readiness, final DVFA scorecard with delta tracking
5
Synthesis
Conductor
Say this
"Conductor, synthesize the full engagement. Here's what each agent found: [paste summaries of key findings from each agent]. Surface the conflicts, produce the integrated DVFA score, and give me strategic recommendations."

Conductor identifies agreements, surfaces where agents disagree, produces the unified strategy with DVFA scoring.

Output: Integrated Strategy Report with DVFA scores, conflicts, recommendations
6
Deliverables
Publisher
Say this
"Publisher, create [executive summary deck / strategy report / one-pager / workshop guide] from the Conductor's synthesis. Audience: [C-suite / innovation team / board]."
Output: Board-ready artifacts (.pptx, .docx, .html, .pdf)

What to Paste Forward

You don't need full outputs from every agent. Carry forward: the focal question, 3-5 key findings per agent, specific conflicts or surprises, and DVFA evidence. Skip the methodology walkthrough.

When to Skip Agents

Not every engagement needs all 13. Typical Standard engagement: Scout + Empathy + Architect (Stage 2), 1-2 intersection agents (Stage 3), Scorekeeper + 1 ops agent (Stage 4), then Conductor + Publisher.

Stage Gate Verification

After each stage, the Conductor verifies: Did agents stay in scope? Does output align with the focal question? Is evidence real or assumed? Any cross-agent contradictions? Actions: PASS, FLAG, or BLOCK.

Agent Quick Reference

Navigator - Intake, scoping, maturity assessment
Publisher - Decks, reports, workshop materials
Scout - IFTF foresight: signals, drivers, scenarios
Empathy - Personas, journeys, desirability testing
Architect - System maps, DOC, leverage points
Visionary - Future personas, speculative design
Integrator - Pilot-to-production, scalability
Sentinel - Stress testing, resilience mapping
Radar - Competitive intel, patent scanning
Banker - H1/H2/H3 portfolio balance
Scorekeeper - DVFA scoring, maturity metrics
Bridge - Change readiness, resistance mapping
Conductor - Cross-agent synthesis, unified strategy
Tip - Say "review as [codename]" for any agent's perspective
